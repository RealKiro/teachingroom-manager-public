# 教室设备管理系统工作记录

[English](./WORK_LOG_2026-05-18.en.md)

日期： 2026-05-18

## 项目背景

本项目最初以一份内部教室设备 Excel 台账为原型，将分散在表格中的教室设备数据整理为一个可持续维护的 Web 管理系统。真实台账现已移出仓库，改由 `初始化数据表格（虚拟）.xlsx` 提供无真实信息的初始化示例。

目标不是简单替代 Excel，而是把教室信息作为校内基础数据源，支持日常巡查、变更审核、Excel 导入导出，以及其他部门系统连接使用。

## 原始需求

1. 使用 Web 方式管理所有教室信息，支持手机、电脑、平板访问。
2. 支持快速呈现和条件筛选。
3. 支持用户角色管理，包括管理员和巡查员。
4. 巡查员可以提交数据变更。
5. 管理员需要对变更做二次审核，通过对比新旧数据后才更新正式数据。
6. 支持输出 Excel 文档，用于日常工作。
7. 字段先依据源 Excel 设置，后续可增加新字段。
8. 用户数较少，最多约 10 人，系统应保持轻量、易部署、易维护。
9. 参考 DNS 管理项目的显示形式，做成清爽的本地 Web 工具。

## 需求调整

- 一个教室通常有前门和后门，因此增加 `前门门牌号` 和 `后门门牌号`。
- “朝向”改为“楼侧”，例如 `A栋 · 南侧`。
- 楼侧字段保留在数据中，但当前不作为顶部快速筛选条件。
- 教室数据应作为底层基础数据，其他部门可以快速连接使用。
- 字段管理入口不放在日常页面里，新增字段由后台维护。
- 用户管理只对超级管理员 `admin` 可见。
- 常用字段在变更弹窗中提供全库已有内容建议。
- Excel 上传后进入审核流程，不直接覆盖正式数据。
- 手机端布局需要适配现场巡查。
- 超级管理员需要查看操作记录和房间历史更新情况。

## 已实现功能

### 数据导入

- 首次启动自动读取源 Excel。
- 已导入 A/B/C 三栋共 92 间教室。
- 原始 A/B/C 三个 Sheet 被整理为统一数据结构。

### 教室字段

核心字段包括：

```text
楼栋
楼侧
教室编号
前门门牌号
后门门牌号
班级/用途
现有电子屏
书写板类型
扩音类型
安装日期
级部
计划电子屏
计划书写板
计划扩音
巡查备注
```

### 页面管理

- 桌面端使用表格视图。
- 手机和平板端使用卡片视图。
- 顶部支持搜索、楼栋、级部、计划、审核状态筛选。
- 手机端筛选区、统计区和教室卡片已做紧凑化。

### 权限和角色

公开版首次账号：

```text
超级管理员用户名：admin
密码：由 INITIAL_ADMIN_PASSWORD 提供，或首次启动时随机生成
```

说明：

- 不内置固定密码，也不自动创建巡查员账号。
- 未设置初始密码时，可从 `data/initial-admin-password.txt` 读取一次性登录密码。
- `role=admin` 表示具备数据审核权限。
- 只有用户名为 `admin` 的账号被视为超级管理员。
- `admin` 账号不能被停用，也不能被降级。

### 用户管理

超级管理员可进行：

- 查看全部用户
- 新增巡查员或管理员
- 修改用户姓名
- 修改用户角色
- 启用或停用用户
- 重置用户密码
- 删除用户
- 查看提交和审核记录数量

删除用户采用软删除，历史记录保留。

### 操作记录

超级管理员可查看：

- 登录和退出
- Excel 导出
- Excel 上传生成审核
- 提交变更
- 审核通过或拒绝
- 用户新增、修改、重置密码、删除
- 数据库备份和恢复

支持关键字、操作类型和操作用户筛选。

### 审核流程

- 用户提交变更后不直接更新正式数据。
- 系统生成待审核变更。
- 管理员查看字段级新旧值对比。
- 审核通过后才写入正式教室数据。
- 审核拒绝后保留记录。

### Excel 导入导出

- 支持按当前筛选结果导出 Excel。
- 导出的 Excel 按楼栋分 Sheet。
- 支持上传修改后的 Excel 并生成待审核变更。
- 兼容系统导出的单行表头 Excel 和原始 A/B/C 栋 Excel 格式。

### 基础数据 API

已增加只读开放 API，供其他部门系统连接。

接口：

```text
GET /api/open/meta
GET /api/open/fields
GET /api/open/summary
GET /api/open/classrooms
GET /api/open/classrooms/:id
```

特点：

- 只读
- 使用 `X-API-Token` 访问
- 支持 CORS
- 不绕过审核流程

### 登录会话持久化

- 登录 session 保存到 SQLite。
- 新增 `user_sessions` 表。
- 服务启动和运行期间会清理过期 session。
- 避免 Express 默认 `MemoryStore` 生产环境警告。

### 数据库备份和恢复

- 新增超级管理员专用的 `数据库备份` 页面入口。
- 系统自动创建每日数据库备份。
- 超级管理员可以手动创建、下载、上传并启用备份。
- 启用前校验数据库完整性、必要数据表和 `admin` 账号。
- 启用前自动创建 `恢复前备份`。
- 启用后服务自动重启。

## 技术方案

- 后端：Node.js + Express
- 数据库：SQLite
- Excel：ExcelJS
- 前端：原生 HTML/CSS/JavaScript
- 部署：Node.js + systemd 或 Docker Compose

选择该方案的原因是用户数量小、部署环境简单、维护成本低，并且 SQLite 文件便于备份、迁移和恢复。
