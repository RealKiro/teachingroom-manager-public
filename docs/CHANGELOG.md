# 更新日志

[English](./CHANGELOG.en.md)

## 2026-07-18

### 公开脱敏版

- 从无历史文件快照建立独立公开仓库，不继承私有仓库提交、分支或 PR 引用。
- 移除固定默认密码；首次启动只创建 `admin`，密码由 `INITIAL_ADMIN_PASSWORD` 提供或随机生成到受保护的运行时文件。
- 不再自动创建巡查员账号；管理员首次改密后自动删除临时密码文件。
- 移除私有仓库地址、本机绝对路径和其他仅适合内部交接的元数据。

### 虚构初始化数据

- 从仓库当前版本删除真实教室设备工作簿，替换为 `初始化数据表格（虚拟）.xlsx`。
- 新工作簿包含使用说明和 12 条完全虚构的教室记录，覆盖系统当前支持的 18 个标准字段。
- 首次启动、Docker 挂载、中英文部署文档和 API 筛选示例已同步使用虚构数据。

### 文档语言拆分

- 中文和英文不再写在同一份 Markdown 文件中。
- 无语言后缀的文件作为默认中文版本，英文版本统一使用 `.en.md` 后缀。
- 每对文档增加双向语言链接，并新增中英文文档中心。
- 中英文版本保持相同的章节结构、命令示例和功能范围。

## 2026-07-13

### 自动备份上限

- 取消备份保存天数限制；自动备份永久保留至 200 份，超过后删除最旧文件，手动备份和恢复前备份不自动删除。

## 2026-07-12

### 全系统复核与加固

- 登录 session 增加用户索引；用户停用、降级、删除、重置密码或自行改密后，相关旧 session 立即失效，并且每次受保护请求都会重新读取当前角色。
- 普通管理员与巡查员的新增教室、字段修改、照片上传和照片删除全部进入交叉审核，提交人不能审核自己的申请。
- 待审核字段增加冲突检测，审核通过前再次核对正式旧值，避免旧申请覆盖后来已经生效的数据。
- 新增统一时间线回退，可逆序处理字段、教室新增和照片操作；旧记录缺少逆向数据时会安全阻止并提示使用数据库备份。
- Excel 导出增加隐藏的字段定义工作表，巡查备注及后续新增字段可以导出后再上传，并继续生成审核申请。
- 弱网提交增加 IndexedDB 持久队列、请求幂等键、在线状态和自动退避重试；审核、回滚和恢复等高风险动作不进入队列。
- 操作记录改为完整结果筛选后分页，不再只搜索最近 500 条；照片和统一回退记录纳入筛选。
- 开放 API 只发布显式允许字段，仅接受请求头令牌，默认关闭 CORS；照片上传校验格式和文件头，并阻止 SVG/HTML 同源执行。
- 自动备份增加保留天数、数量上限和可选镜像目录；Docker Compose 持久化数据库、备份、上传和导出目录。
- 每次服务启动都会执行备份保留策略，即使当天已有自动备份也会清理超出数量或保留期的旧自动备份。
- 数据库恢复完成后清空备份中携带的全部登录 session，防止旧会话随恢复重新生效。
- 桌面表格重新分配列宽，手机统计字号恢复为 23px，照片说明和楼侧编辑显示与审核规则一致。
- 新增真实 SQLite 集成测试，覆盖交叉审核、会话失效、幂等、冲突、照片、统一回退预览、开放 API、审计分页和 Excel 往返；依赖审计为 0 个已知漏洞。

## 2026-06-23

### 用户自助改密

- 顶部账号区域新增 `修改密码` 入口，所有已登录用户可用。
- 修改密码需要输入当前密码、新密码和确认新密码。
- 服务端会校验当前密码是否正确，新密码至少 6 位。
- 用户自己修改密码会写入操作记录，便于超级管理员审计。
- 超级管理员在用户管理中重置他人密码的功能保持不变。

### P1-P3 问题修复

- 超级管理员可在操作记录中对 `新增审核通过` 记录执行 `撤销此新增` 或 `还原到此记录之前`。
- 新增教室回滚会先检查该教室是否已有后续变更或照片，避免删除时破坏历史记录。
- 待审核统计现在同时包含普通字段变更和新增教室申请。
- Excel 上传遇到未匹配的新教室时，管理员上传会自动生成新增教室待审核申请。
- Excel 上传新增申请会写入逐条审计记录，并保留来源文件。
- 用户管理中的提交次数、审核次数现在包含新增教室申请。
- Excel 导入识别范围从 A/B/C 栋放宽到任意字母开头的教室编号，支持 I、J 等新增楼栋。

### 新增教室审核

- 普通管理员新增教室不再直接写入正式数据，而是生成 `新增待审核` 申请。
- 只有其他管理员可以审核该新增申请；提交人不能审核自己的新增申请。
- 超级管理员 `admin` 保留直接新增教室的权限。
- 新增教室申请通过后才写入正式教室数据，拒绝后不会占用教室编号。
- 新增操作记录类型：提交新增教室、新增审核通过、新增审核拒绝。

### 弱网络新增提交

- 新增教室提交会生成客户端请求 ID，用于识别同一次提交。
- 手机或弱网络环境下，新增教室请求遇到超时或网络异常会自动重试。
- 同一次请求重试不会重复生成教室或重复生成审核申请。

## 2026-06-15

### 巡查照片

- 教室列表在 `暑期计划` 和 `审核` 之间新增 `备注` 列。
- 有巡查备注或照片时，列表显示 `查看` 链接，可弹窗查看备注信息和照片。
- 在教室变更弹窗的巡查备注后新增 `巡查照片` 区域。
- 登录用户可以给教室上传图片，照片会直接保存到 SQLite 数据库中。
- 照片支持缩略图预览、点击查看原图、显示上传人、时间和文件大小。
- 管理员可以删除照片，普通用户可以删除自己上传的照片。
- 照片随数据库备份一起保留，便于恢复和迁移。

### 交叉审核

- 变更审核改为交叉审核，提交人不能审核自己提交的变更。
- 超级管理员 `admin` 保留审核自己提交变更的权限。
- 普通管理员自己提交的待审核项会显示提示，并隐藏通过/拒绝按钮。
- 其他管理员仍可正常审核该变更。

### 管理员新增教室

- 管理员界面新增 `新增教室` 入口，只有管理员角色可见。
- 新增教室表单按当前字段定义动态生成，后续新增字段后可以自动出现在表单中。
- 楼栋和教室编号为必填项，系统会校验同一楼栋下教室编号不能重复。
- 新增记录支持前门、后门、级部、现有设备、暑期计划和巡查备注等字段。
- 新增教室会写入操作记录，便于超级管理员后续查询。

## 2026-06-05

### Excel 导出说明

- `导出说明` 页新增 `导出范围`、`是否筛选` 和 `筛选条件`。
- 无筛选导出显示 `全部数据`，筛选或部分列表导出显示 `筛选结果`。
- `导出说明` 页新增导出范围内的统计结果，包括记录数量、涉及更新、待审核、楼栋分布和更新项目统计。
- 统计结果仅基于本次导出的记录，方便日常工作中直接确认当前 Excel 文件范围。

## 2026-05-19

### 数据库备份和恢复

- 新增超级管理员专用的 `数据库备份` 入口。
- 系统启动后会检查当天是否已有自动备份，没有则自动创建一份。
- 系统运行期间每小时检查一次当天自动备份状态，避免漏备份。
- 备份文件保存到项目 `backups/` 目录，文件名包含北京时间时间戳和备份类型。
- 超级管理员可以手动创建当前数据库备份。
- 超级管理员可以下载任意备份文件到本地。
- 超级管理员可以直接启用服务器上的某个备份。
- 超级管理员可以上传 SQLite 数据库文件并启用。
- 启用备份或上传数据库前，系统会校验 SQLite 完整性和必要数据表。
- 启用前会自动创建一份 `恢复前备份`，防止误恢复后没有退路。
- 数据库启用后服务会自动重启，恢复操作会写入新数据库的操作记录。

### 超级管理员回滚

- 新增“撤销此修改”：超级管理员可以在操作记录中撤销某一次已审核通过的修改。
- 新增“还原到此记录之前”：超级管理员可以把教室基础数据还原到某条已审核记录发生之前的状态。
- 回滚执行前会先预览影响范围，包括涉及教室数、字段数和字段级当前值/恢复值。
- 单次撤销如果发现同一字段已经发生后续变化，会提示冲突，避免覆盖后续有效修改。
- 如果回滚目标字段存在待审核变更，系统会阻止回滚，要求先处理待审核项。
- 回滚操作会写入操作记录，保留可追溯记录。

### 北京时间显示

- 页面中的审核时间和操作记录时间统一按 UTC+8 北京时间显示。
- 数据库继续按 UTC 保存时间，前端显示时转换为 `Asia/Shanghai`。
- Excel 导出说明页中的导出时间也固定使用北京时间。

### 登录会话持久化

- 将登录 session 从 Express 默认内存存储切换为 SQLite 存储。
- 新增 `user_sessions` 数据表，用于保存会话内容、过期时间、创建时间和更新时间。
- 服务启动时会自动清理已过期 session。
- 服务运行期间每 30 分钟清理一次已过期 session。
- 用户主动退出登录时，会同步删除对应 session。

影响：

- 服务器重启后，未过期的登录会话可以继续保留。
- 不再出现 Express `MemoryStore` 生产环境警告。
- 部署仍然只依赖 Node.js 和 SQLite，不增加 Redis 等外部服务。
